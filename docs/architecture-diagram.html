agent-tui Architecture

Terminal UI scaffold for AI coding agents — Protocol-driven, event-streaming, HITL approval

TUI Layer (Textual)

  • • AgentTuiApp — main Textual app
  • • 18 widget files (ChatInput, Messages, etc.)
  • • 21 slash commands with autocomplete
  • • Virtualized message store for long chats
  • • Background workers for non-blocking startup
  • • Theme switching, model selector, thread browser

Agent Backends

  • • AgentProtocol — clean interface contract
  • • StubAgent — mock for dev/testing
  • • DeepAgentsAdapter — real agent via deepagents SDK
  • • EventTranslator — LangGraph → AgentEvent mapping
  • • LocalShellBackend + SandboxBackend
  • • MCP server management (stdio-based)

Security & HITL

  • • ApprovalMenu — Yes/Auto/No for tool calls
  • • AskUserMenu — interactive question flow
  • • InterruptOverlay — mid-execution approve/edit/reject
  • • Dangerous shell pattern detection (14 patterns)
  • • File path allowlisting (DEEPAGENTS_ALLOWED_DIRS)
  • • Skill path containment validation

Persistence & State

  • • SQLite (aiosqlite) — session checkpoints
  • • LangGraph MemorySaver integration
  • • Thread management (list, delete, resume)
  • • UUID7 for time-ordered thread IDs
  • • Message count caching with freshness tokens
  • • Session stats (tokens, time, turns)

External Integrations

  • • OpenAI / Anthropic / Google / NVIDIA APIs
  • • Tavily web search + httpx URL fetch
  • • MCP servers via .mcp.json config
  • • PyPI version checking (auto-update)
  • • ripgrep (rg) as faster grep fallback
  • • AGENTS.md memory sources (user + project)

Skill System

  • • 7-tier filesystem discovery with precedence
  • • built-in → user → project → .agents → .claude
  • • YAML front-matter stripping
  • • Symlink traversal protection
  • • AGENT_TUI_EXTRA_SKILLS_DIRS env var
  • • Skill invocation with metadata envelopes
